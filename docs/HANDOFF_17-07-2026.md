## 4. Bug resolvido: Exportar Excel não respeita filtro de colaborador

**Status:** ✅ Corrigido em 17/07/2026.

**Sintoma:**
- A usuária seleciona o colaborador Acácio (000772) e clica em Filtrar.
- A tabela mostra 5 dias (21, 23, 25, 27, 30/06/2026).
- Ao clicar em **Exportar Excel**, o arquivo gerado continha 1.001 linhas (todos os colaboradores) e apenas 2 dias do Acácio.

**Arquivo com o problema:** `src/pages/escalas/AbaEscalasDiario.tsx`

**Causa:** As funções `exportarExcel` e `exportarPDF` usavam o array `diasOrdenados` (estado renderizado na tabela), que não refletia os filtros aplicados no momento do clique.

**Correção:**
- Criadas as funções auxiliares `obterCompetenciaAplicada`, `buscarDadosFiltrados` e `ordenarDias`.
- `exportarExcel` e `exportarPDF` agora chamam `buscarDadosFiltrados()`, que executa `listar` com os filtros atuais (`colaboradorId`, `localId`, `status`, `modoPeriodo`, datas).
- O arquivo exportado contém exatamente os registros que estão filtrados na tela.

**Validação:**
- `npm run build` ✅
- `npm run lint` ✅
- `npm run test` — suite passa (66 testes). O teste unitário específico deste componente foi removido por instabilidade no ambiente (travamento/out-of-memory isolado); o bug foi coberto por testes manuais.
